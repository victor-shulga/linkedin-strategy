# Методологія Content Engine

Дистиляція курсу «LinkedIn Social Selling» Віктора Шульги (Модулі 1–6) + перевірені дані
зі стратегії агенції (Cowork, 2026-06). Це **нормативний документ**: усі кроки engine
читають звідси правила, а не вигадують свої.

---

## 1. Стратегічні фреймворки (Module 1)

### 1.1 Три можливі цілі (визначають weekly flow)
1. **Brand awareness** — стати відомим у ніші.
2. **Thought leadership + нетворк** — інсайти, історії клієнтів, експертні думки.
3. **Ліди і ревеню** — вхідні запити + діалоги.

### 1.2 ICP: складові
Цільові індустрії → фірмографіка (гео, розмір/дохід) → цільові сервіси → enrichment →
цільова персона → тіри. Дві головні дії для створення: **аналіз CRM** (клієнти за 3–5 років →
групування по індустріях → топ 1–2) і **CustDev** (топ-10 клієнтів, дзвінок, не листування).

### 1.3 Чотири аудиторії комунікації
- **ICP** — хто платить.
- **Peers** — вирішують ті самі проблеми без бюджетів (двигун охоплень і комент-мережі).
- **Co-pilots** — продають інші сервіси тій самій аудиторії (партнерства, рефералки).
- **Thought Leaders** — великі акаунти з твоєю аудиторією.

### 1.4 Сервіс → Use case → Offer (9-польова картка оферу)
1. Назва сервісу · 2. Use case · 3. ICP · 4. Які проблеми вирішує · 5. Deliverables ·
6. Capabilities · 7. Dream outcome · 8. Цінова модель · 9. Тривалість.
Портфоліо оферів: безкоштовні матеріали → лоу-тікет → фронтенд → хай-тікет.

### 1.5 Фреймворк «ворога»
Ворог ≠ конкуренти. Ворог = ситуація/застарілий спосіб ведення бізнесу, що тримає клієнта
в поточному стані. 4 питання: яку проблему вирішуєте → як інакше → хто ворог → як перемагаєте.
Диференціація підходу: методологія / фреймворки / унікальний процес («секретний соус»).

### 1.6 Тема і підтеми
- Формула теми: **{ключова компетенція} для {ICP}**. Тільки ОДНА тема.
- Підтеми = частини ключових сервісів (кожна вирішує свою проблему) + 1 персональна.
- Чим менше підтем, тим краще (≤ 5–7).

### 1.7 KPI + commitment
KPI завжди з часовими рамками. Commitment (зобов'язання) — фундамент реалістичності:
N постів/тиждень, M хв engagement/день, K доданих ICP/день.

## 2. Контент-стратегія (Module 3)

### 2.1 Три критерії робочої ідеї (фільтр Idea Pool)
1. **Хук > ідея.** Сильний хук + слабкий пост >> слабкий хук + сильний пост.
2. **Компетенція.** Тільки ідеї, що підкреслюють компетенцію або будують довіру.
3. **Унікальний фід.** «Якщо всі постять це — у моєму фіді цього бути не повинно».

### 2.2 Фільтр-питання для кожної ідеї
- Чому я достатньо кваліфікований цим ділитись?
- Чи відповідає це моїм цілям?
- Чи є докази компетенції?

### 2.3 Формула хука
**Ідея + Компетенція = Чудовий хук.** Якщо немає доказів/результатів — 3 запасні стратегії:
(а) ресьорч, що підтверджує думку; (б) «я робив ось це — що вийшло» / «ось що я роблю»;
(в) своя історія (аутентичність без великих цифр).

### 2.4 Маппінг тип ↔ воронка
| Тип | Воронка |
|---|---|
| Персональні історії | TOFU |
| How-to, How-I, каруселі, giveaways | MOFU |
| Case study, hand raiser | BOFU |

### 2.5 Тижневий ритм (дефолт, перевизначається у стратегії клієнта)
| День | Воронка | Формат | Тип |
|---|---|---|---|
| Пн | TOFU | Single image | Персональна історія |
| Вт | MOFU | Carousel / Giveaway | How-to, How-I |
| Ср | BOFU* | Single image | Case study / Giveaway / Hand raiser |
| Чт | MOFU | Single image / Carousel | How-to, How-I, Contrarian |
| Пт | TOFU | Single image / Video | Персональна історія |

\* BOFU не щотижня: чергування Wed MOFU (непарні тижні) / BOFU (парні) — з реальної стратегії
агенції. Вихідні — не постимо. Максимум 1 пост/день.

### 2.5а Вісім обігравань однієї ідеї (Module 3, крок 7)
Одна ідея = до 8 кутів подачі: **How-to · How-I · Відео · Карусель · Лід-магніт · Villain post ·
Помилки · Вивчені уроки.** Використовується: (а) при розмноженні ідеї-переможця у 08-repurpose,
(б) коли рекомендатор пропонує варіанти на слот — та сама ідея може зайти іншим кутом.

### 2.5б Свайп-файл референсів (steal like an artist)
Research-крок зберігає в картку Idea Pool не лише лінк на outlier-пост (Source link), а й
**скрін креативу-референсу** (в body картки) — щоб на продакшні було видно, ЯК це було подано,
а не лише про що. Переробка завжди під свій ICP і voice, ніколи не калька.

### 2.6 Реюз і рециркуляція
- Одна ідея реюзається з паузою **1–3 місяці**.
- Перепост старого поста-переможця — через **≥ 3 місяці** (текст можна оновити).
- Тактика «Ріжемо-Розширюємо»: лонгформ → пости/каруселі; вдалий пост → лонгформ.
- Тактика «1 ідея = 6 постів»: ідею-переможця прогнати через матрицю типів постів.

## 3. Написання постів (Module 4)

### 3.1 Модель уваги
Стоп-скрол робить **креатив** → потім читається **трейлер** (хук + рехук) → потім «…more».
Креатив головніший за текст.

### 3.2 Копірайтинг-фреймворки (вибір за типом поста)
- **PAS** (Problem–Agitation–Solution) — прогрів, контент про болі. → How-to, Contrarian.
- **BAB** (Before–After–Bridge) — кейси, демонстрація результатів. → Case study, How-I.
- **AIDA** — продажні, промо, експертні пости. → Giveaway, Hand raiser.

### 3.3 Канонічна структура поста (усі формати КРІМ каруселей)
1. **Трейлер** (хук + рехук) → 2. **Проблема** (+ підсилення) → 3. **М'ясо** → 4. **Резюме** (кінцівка + CTA).
Резюме: читач має змогти прокоментувати, не перечитуючи весь пост.

### 3.4 Рехук — 3 варіанти
Розкрити думку хука / зачеленджити хук / міні-CTA («збережи на потім»).

### 3.5 Форматування
Короткі речення · пробіли («текст дихає») · «ялинки» для сканування · ритм · нумеровані списки.
Target: **400–500 слів** (58.3% віральних). < 100 слів = 12.4%. Не скорочувати «для зручності» —
LinkedIn винагороджує dwell time.

### 3.6 Структура каруселі (7 блоків — ЖОРСТКА, кожен блок обов'язковий)
1. **Обкладинка** — всі складові хука; задача — стоп-скрол.
2. **Контекст** — навіщо карусель, чого очікувати (обов'язковий: текстового поста немає).
3. **М'ясо** — правило: **одна думка = один слайд**.
4. **Результати** — докази: скріншоти метрик, відповіді клієнтів.
5. **Бонус** — подарунок тим, хто догорнув (привчає читати до кінця).
6. **CTA** — попросити дію.
Текст до каруселі = однорядковий хук, НЕ повноцінний пост. Каденс: ≈ 1 карусель на 2 тижні,
тільки коли тема структурно вимагає.

### 3.7 Кейс-стаді пост (BOFU) — 8 елементів
1. Трейлер (формула: **результат + для кого**) · 2. Проблема · 3. Підсилення · 4. Рішення ·
5. Результати · 6. Рефреймінг на читача · 7. CTA + офер · 8. Креатив = доказ.

### 3.8 Чекліст написання (6 кроків)
Фреймворк → трейлер (кожен рядок веде до наступного?) → проблема → м'ясо → резюме → форматування.

## 4. Дистрибуція (оновлено 06.2026 за Algorithm Insights Report 2026 — van der Blom, 1.3M постів)

**Рамка: Interest Graph замість Relationship Graph (з осені 2025).** Раніше пост показувався
переважно 1st-degree конекшенам; тепер алгоритм роутить за релевантністю теми: лише 31% фіду —
від конекшенів (було 72% у 2024), 33%+ — від людей, що ніколи не фоловили автора; Suggested
Posts 5%→9% фіду за рік. Наслідки: кількість фоловерів важить менше за Topic Authority;
сильний topic-сигнал = +40% життя поста; акаунт грейдиться кожні 90 днів за topic consistency —
якщо LinkedIn не може автора категоризувати, він не може його дистриб'ювати. Тому вузький ICP
досяжний через інтерес-кластер навіть без підписки, а пілари клієнта мають сидіти на ОДНІЙ темі.

1. **−20 хв до посту:** 5 змістовних коментарів (15+ слів) під постами свого topic-кластера →
   ~90% видимість поста цим акаунтам.
2. **Час:** Європа 10:00–14:00 за таймзоною аудиторії; таймінг — у бакеті «20.5% поза контролем»,
   важить менше за nurturing. Нд — найсильніший день для personal storytelling.
3. **Перша година ≈ 50% охоплення.** Відповідь на кожен комент ≤ 30 хв (+64% коментів,
   2.3× переглядів); протягом 2 год — 2 власні коменти під своїм постом. Nurture перші 4 год =
   10–14× охоплення vs post-and-ghost.
4. **Self-repost через 4–6 год, рівно один раз:** +12–24% охоплення, +2 дні життя. Ніколи двічі.
5. **Лінки:** «лінк у першому коменті» МЕРТВИЙ (коменти з лінками ріжуться; конверсія −3.4×).
   Пост на конверсію → 1 лінк у body (~−19% reach). Пост на охоплення → без лінків.
6. **⚠️ Comment-to-get = engagement bait з 2026** («напиши X в коментах» → shadowban-ризик,
   AI-модерація). Доставка лід-магнітів: «напиши в DM» / лінк у body / Lead Gen Form
   (конверсія 10–13%). DM-ask — дефолт: зберігає особистий діалог для кваліфікації.
7. **0 хештегів, 0 тегів** (макс 3 згадки; теги без відповіді — мінус).
8. **Saves + reposts** — єдині сигнали, що воскрешають пост після 48 год → CTA «Зберігайте собі»
   підсилювати. Коменти частково девальвовані (поди/AI-репліки); лайки з інтерес-кластера знову важать.
9. **Макс 1 пост/день** (кілька: +18% сумарно, −26% на пост). 3 пости на 2 теми > 5 постів
   на 6 тем — **Topic Authority: 50.1% охоплення визначено ДО публікації** (профіль-бейзлайн +
   топік-консистентність; зміна тем/форматів навмання = −20–30%).
10. Комент-машина: 5–15/день, мікс 20% ICP / 50% peers / 30% інфлуенсери; тільки змістовні
    15+ слів, без ШІ-коментарів. Конекти 5–10/день; видаляти неприйняті > 14 днів.
11. Перформанс дивитись на день 3/7/10 (не лише 48 год); republish ≥ 90 днів; конверсія
    формату 45–60 днів. Оптимум довжини за RvdB: 1 400–1 800 знаків [конфлікт з Cowork
    400–500 слів — тестувати на власних даних].

## 6. KPI і фідбек-цикл (Module 6)

### 6.1 Піраміда метрик
NSM (напр. New Revenue) → L1 Revenue (Total/New Revenue, ACV, Win Rate, Pipeline Velocity) →
L2 Sales (Won Deals, Sales Cycle, SQLs: outbound/inbound/referral, Appointments, Intro Calls) →
**L3 LinkedIn**: Impressions · Follower Growth · Engagement Rate · Profile Views · Messages Sent ·
**Leads Generated (лід = розпочатий діалог)** · Calls (MQL) · Generated Pipeline $.

### 6.2 Каденси
- **Щотижня:** інжест метрик постів (L3), оновлення Posts DB.
- **Щомісяця:** аналіз постів → теми-переможці → рециркуляція (3 тактики з §2.6); перегляд SSI.
- **Раз на 1–3 місяці:** реюз ідей-переможців; репост ≥ 3 міс.

### 6.3 Правило фідбеку
Контент-стратегія покращується на основі **(а) залучення І (б) потоку лідів** — не лише лайків.
Тема, що дає діалоги з ICP, цінніша за тему, що дає охоплення від peers.

## 7. DM-машина (Module 5; контекст для CTA і lead magnet флоу)

Сигнал → діалог → кваліфікація → допомога → дзвінок → продаж.
- Єдина ціль першого повідомлення — **відповідь** (не зустріч, не продаж).
- < 50 слів · мобайл-перший · you-language · релевантність > персоналізація · макс 3 спам-слова.
- Кожне повідомлення закінчується питанням: qualifying → probe → snatch.
- Шаблон доставки лід-магніта: {подяка за комент} {лінк} {qualifying питання}.
- «Гроші не в охопленнях і лайках — а в діалогах».
